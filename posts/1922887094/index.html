HEXO-搭建个人博客（一）| 搭建基础环境

发表于2023-04-14|更新于2023-04-17
|阅读量:

安装Git

官网下载：https://git-scm.com/

我用的版本是：2.33.0.windows.2

安装Node.js

官网下载：http://nodejs.cn/

我用的版本是：v16.15.0

安装Hexo

在cmd窗口输入指令

npm install -g hexo-cli

在磁盘创建一个文件夹“myblog”，在这个文件夹下右键git bash打开

初始化hexo

hexo init

启动hexo

hexo s

根据提示复制网址打开

http://localhost:4000

将Hexo部署到GitHub

在GitHub上创建个人仓库

1. 创建一个GitHub账号
2. 建立仓库，命名为“用户名.github.io”, 例： Demon-dou.github.io

生成SSH添加到GitHub上

在cmd窗口输入

git config --global user.name “yourname”

git config --global user.email "youremail"

例如：
git config --global user.name Demon-dou
git config --global user.email 705619743@qq.com

注：
yourname: GitHub的用户名
youremail：注册GitHub的邮箱

可以通过以下两个命令进行查询你的用户名和邮箱
git config user.name
git config user.email

接着在cmd窗口中输入下方的命令，然后敲击三下回车

ssh-keygen -t rsa -C 705619743@qq.com

注：
后面的是自己的邮箱

执行完命令后会自动在C的目录下生成.ssh目录，在窗口界面中可以看到路径

打开.ssh目录，id_rsa.pub是公钥，另一个私钥，复制公钥中的内容

进入到GitHub，点击setting中的SSH，将密钥粘贴进去，点击“Add SSH Key”

部署到GitHub中

修改根目录的配置文件

打开_config.yml文件，滑动到最后
修改内容：
deploy:
  type: git
  repo: git@github.com:Demon-dou/Demon-dou.github.io.git
  branch: main
  
注：获取repo的方法
在GitHub上自己的仓库中复制SSH

在自己的根目录中，打开git，输入命令，为了安装deploy-git

npm install hexo-deployer-git --save

依次执行下面三个命令

hexo c     # clean的缩写，为了清除缓存文件
hexo g	   # generate的缩写， 为了生成网站静态文件
hexo d	   # deploy的缩写，为了部署到设定的仓库

现在就可以通过“用户名.github.io“进行访问你自己的博客了

关于Hexo根目录中的各个文件夹的解释

文件夹 解释
node_modules node.js模块
scaffolds 文章模板
source md源码
themes 主题源码
