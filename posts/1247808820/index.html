HEXO-搭建个人博客（五）| 优化文章链接

发表于2023-04-20|更新于2023-04-20
|阅读量:

待优化问题

Hexo默认使用的链接格式是：

# permalink: :year/:month/:day/:title/

使用这种格式会将文章标题展示在链接地址上，若更改文章的名字，就会导致链接失效

变量及其含义

变量 描述
:year 文章的发表年份（4 位数）
:month 文章的发表月份（2 位数）
:i_month 文章的发表月份（去掉开头的零）
:day 文章的发表日期 (2 位数)
:i_day 文章的发表日期（去掉开头的零）
:hour 文章发表时的小时 (2 位数)
:minute 文章发表时的分钟 (2 位数)
:second 文章发表时的秒钟 (2 位数)
:name 文件名称
:title 文件名称 (relative to “source/_posts/“ folder)
:post_title 文章标题
:id 文章 ID (not persistent across cache reset)
:category 分类。如果文章没有分类，则是 default_category 配置信息。
:hash SHA1 hash of filename (same as :title) and date (12-hexadecimal)

解决办法

  • 使用abbrlink插件
  • https://github.com/rozbo/hexo-abbrlink

安装abbrlink插件

在博客根目录下执行以下命令：

npm install hexo-abbrlink --save

编辑Hexo的配置文件_config.xml

修改配置文件如下：

#permalink: :year/:month/:day/:title/
#permalink_defaults:
permalink: posts/:abbrlink/
abbrlink:
  alg: crc32 #support crc16(default) and crc32
  rep: dec   #support dec(default) and hex

重新部署

使用以下命令，会在博客源文件中看到abbrlink

hexo clean
hexo g
hexo new "新博客的名字"
hexo d
-- 通过浏览器查看（会发现链接地址变了）

优化后的优点

  • 纯英文和数字链接非常有利于SEO
  • 只要abbrlink编号不变，该文章的url就不变，可以随意更改文件名
